# Open ScholarPeer — Developer Agent Instructions

This `AGENTS.md` provides context and conventions for AI agents helping to **develop and maintain this library**. It is not used by end users running OSP — they get a separate `AGENTS.md` written into their project root by `install_copilot.sh`.

---

## Project Overview

Open ScholarPeer (OSP) is an open-source implementation of the paper *"ScholarPeer: A Context-Aware Multi-Agent Framework for Automated Peer Review"*. OSP runs as a portable set of skills, slash commands, and MCP tools that install into the user's project directory across five AI tools.

**Repository purpose:** Provide the installable source files — prompts, skills, rules, MCP server — and the sync infrastructure that keeps per-tool adapters consistent.

## The Golden Rule

**Edits go in `extensions/_shared/`.** Never edit the per-tool adapter directories directly — they are wiped and regenerated on every sync.

```bash
# Standard development cycle

$EDITOR extensions/_shared/skills/osp-summary-agent/SKILL.md   # 1. edit canonical
python3 scripts/sync_adapters.py                                # 2. regenerate adapters
python3 scripts/test_parity.py                                  # 3. validate
bash scripts/test_install.sh                                    # 4. smoke-test installers
```

## Key conventions

### Naming

- **Slash commands:** `0..6-osp-<step>` (numbered) + `open-scholar-peer` (stateless dispatcher).
- **Skills:** `osp-<persona>` (no number — order doesn't matter for skills).
- **Brain root:** `.brain/` at user project root.
- **MCP runtime root:** `.scholar-peer/mcp/` at user project root.

### Capability matrix (encoded in `sync_adapters.py`)

| Tool | Subagent | Q&A mode |
| --- | --- | --- |
| Claude Code, Cursor, Gemini CLI, Copilot CLI | yes | subagent (preferred) |
| Antigravity | no | self-reflection fallback (turn markers) |

### Hyperparameter enforcement

Host tools don't expose temperature or other LLM hyperparameters to slash commands. The paper's key parameters (k=3 literature rounds, N_QA=10 probing pairs) are enforced **structurally** via file templates, not numerically. See `defaults/qa_pair_template.md` and `defaults/round_strategy_template.md`.

### Universal artifact structure

Every `.brain/raw/*.md` artifact has three required sections:
- `## Method` — what the agent did (queries run, tools used)
- `## Output` — the actual content
- `## Provenance` — sources cited

This is documented in `docs/ARTIFACT_CONTRACTS.md` and is part of the I/O contract.

## File safety rules

- `.brain/` is gitignored — may contain confidential paper content. Never commit.
- `.scholar-peer/` is gitignored — per-project venv. Never commit.
- The install scripts write only to the user's target directory, never to the source repo (unless running from within the repo itself).
- `merge_mcp_config.py` preserves unrelated user keys when merging into existing MCP configs.

## Common tasks

| Task | Where to start |
| --- | --- |
| Add a new slash command | `extensions/_shared/commands/` + matching skill in `skills/` + sync |
| Refine a persona prompt | `extensions/_shared/skills/<persona>/SKILL.md` + sync |
| Add an MCP provider (e.g. PubMed) | `mcp-server/providers/<name>.py` + register in `osp_mcp.py` |
| Fix a tool's adapter format | `scripts/sync_adapters.py` (capability matrix at top) |
| Update Q&A self-reflection vs subagent logic | `sync_adapters.py` `adapt_qa_body_for_tool()` |
| Update venue criteria fallback | `extensions/_shared/defaults/generic_review_guidelines.md` |

See `docs/CONTRIBUTING.md` for full guidance with checklists.

## What's deferred / out of scope

- **`src/backend`** — DeepAgents JS / LangGraph standalone runtime.
- **`src/frontend`** — Customized Deep Agents UI with stepper UX.
- **Multi-paper sessions** — currently one paper per `.brain/`.
- **PyPI publishing of `osp-mcp`** — currently self-contained venv per project.
- **CI for sync drift** — currently manual; future = pre-commit hook or GH Actions.
